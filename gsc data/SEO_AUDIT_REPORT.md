# Google Search Console Audit Report
## Voxwel.com — Past 28 Days (April 2 – April 29, 2026)

---

## Executive Summary

| Metric | Value | Status |
|--------|-------|--------|
| Total Clicks | 46 | 🔴 Critical |
| Total Impressions | 50,426 | 🟡 High Volume |
| Average CTR | 0.09% | 🔴 Critical |
| Average Position | ~6.2 | 🟢 Decent |

**Core Issue:** The site ranks reasonably well (avg. position 6.2) and generates significant impressions, but **click-through rate has collapsed**. Pages sitting at position 3–5 with 20,000+ impressions are returning **zero clicks**. This is not a ranking problem — it is a **SERP presentation and technical issue**.

## 1. Desktop CTR Collapse (CRITICAL)

### Finding
From `Devices.csv`:

| Device | Clicks | Impressions | CTR | Position |
|--------|--------|-------------|-----|----------|
| Desktop | 32 | 49,828 | **0.06%** | 6.21 |
| Mobile | 13 | 586 | **2.22%** | 21.84 |
| Tablet | 0 | 12 | 0% | 50 |

- Desktop accounts for **98.8% of all impressions** but delivers a **37× worse CTR** than mobile.
- A 0.06% CTR on desktop is catastrophic for any B2B SaaS site.

### Diagnosis
- Desktop users see the site in SERPs but actively choose not to click.
- Likely causes:
  1. **Title tags are truncated or unappealing** on wider desktop SERP layouts.
  2. **Meta descriptions are missing, duplicated, or poorly written**.
  3. **No rich snippets or structured data** appearing on desktop (or appearing broken).
  4. **Slow Core Web Vitals / page speed** causing users to skip the result.
  5. **Competitors have richer SERP features** (sitelinks, ratings, FAQs) that dominate desktop space.

### Action Items
- [ ] Audit title tags for all top-impression pages. Keep under 60 characters. Front-load value props (e.g., "Best EU Whistleblowing Software 2025 | Voxwel").
- [ ] Rewrite meta descriptions for the top 20 pages. Include clear CTAs.
- [ ] Run a desktop-specific SERP preview test to see how snippets render vs. competitors.
- [ ] Check Core Web Vitals (LCP, CLS) specifically for desktop.

### Diagnosis

  3. **Structured data errors** causing the snippet to render poorly.

  5. **AI Overviews or featured snippets** are answering the query above the fold, removing the need to click.

## 3. www vs Non-www Split (HIGH)

### Finding
From `Pages.csv`:
- Line 39: `https://www.voxwel.com/blogs/real-cost-workplace-harassment` (0 clicks, 15 impressions)
- All other 55 URLs: `https://voxwel.com/...`

### Diagnosis
- GSC is treating `www` and non-www as **separate properties** for this page.
- Link equity is being split. Backlinks to `www` and non-www versions are not consolidated.
- This creates **duplicate content** in Google's index.

### Action Items
- [ ] Choose a canonical version (recommend non-www, since 98% of URLs use it).
- [ ] Set up 301 redirects from `www.voxwel.com/*` → `voxwel.com/*`.
- [ ] Ensure the canonical tag on every page points to the non-www version.
- [ ] Update Google Search Console property settings to preferred domain.

## 4. Review Snippet Failure (HIGH)

### Finding
From `Search appearance.csv`:
- Review snippets: **17 clicks / 45,369 impressions = 0.04% CTR**

### Diagnosis
- Review snippets should **increase CTR** by adding star ratings to SERPs.
- A 0.04% CTR means either:
  1. The review snippet is **not rendering visually** in SERPs (structured data error).
  2. The snippet is appearing for **low-intent queries** where users don't click.
  3. Competitors also have review snippets, negating the advantage.

### Action Items
- [ ] Test structured data with Google's Rich Results Test for the homepage and blog pages.
- [ ] Verify review markup uses the correct schema (`AggregateRating`, `Review`, or `Product` / `SoftwareApplication`).
- [ ] Check if review stars actually appear in live SERPs.

## 6. Query Bloat — 1,275 Queries, ~99% Zero-Click (MEDIUM)

### Finding
From `Queries.csv`:
- Total queries tracked: **1,275**
- Queries with 0 clicks: **~1,250+ (98%)**
- Many queries are extremely long, AI-generated sounding questions:
  - *"evaluate the regtech company alphasense on best whistleblower systems (eu directive)"*
  - *"what is an example of 'skimming'? submitting inflated expense claims manipulating payroll systems stealing cash before it is recorded creating fake accounts for payments"*

### Diagnosis
- The site is ranking for a massive number of **long-tail, low-intent queries**.
- These inflate impressions without driving clicks, artificially lowering CTR.
- Possible causes:
  1. Blog content is too broad and ranks for tangential queries.
  2. Google's AI Overviews or Passage Ranking is surfacing content for questions it answers inline.
  3. Bot or automated traffic inflating impression counts.

### Action Items
- [ ] Export the top 100 zero-click queries and check if they align with business goals.
- [ ] If irrelevant, add `noindex` to thin content pages or consolidate posts.
- [ ] Optimize content for **commercial intent** queries (e.g., "buy whistleblowing software", "voxwel pricing") rather than informational only.

## Conclusion

Voxwel does **not have a ranking problem** — it has a **click problem**. The site is visible (position 3–7 for many terms) but fails to convert impressions into clicks. The immediate fixes are:

1. **Rewrite titles and meta descriptions** for the top-impression pages.
2. **Fix the www/non-www split** to consolidate authority.
3. **Investigate the zero-click pages** — something is broken in how they appear in SERPs.
4. **Validate review snippet markup** so stars actually appear.

These are all low-effort, high-impact changes that should significantly improve CTR within 2–4 weeks.
